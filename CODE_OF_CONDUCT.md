# Code of Conduct

## Expected behaviour

- Be respectful in all project interactions.
- Critique code and ideas, not people.
- Assume good faith; ask for clarification before assuming malice.
- Respect privacy. Do not share personal information about contributors or platform members without their consent.
- Stay on topic in issues and pull requests.

## Unacceptable behaviour

- Harassment, including sexualised attention, doxxing, intimidation, or sustained disruption.
- Discriminatory language or imagery targeting protected characteristics.
- Personal attacks.
- Sharing private content from project communications channels without participant consent.
- Brigading or coordinated pressure on individual contributors.

## Scope

This Code applies to all project spaces: the GitHub repository, code review threads, official chat channels, and any in-person or online events organised in the project's name.

It does not govern conversations between members on the dyad platform itself, which are subject to the platform's own community standards.

## Reporting

Report violations to `hello@dyad.berlin`.

- Reports are read by the project maintainers and treated confidentially.
- We aim to acknowledge within five working days.
- We will not retaliate against people who report in good faith.
- If a maintainer is the subject of a report, the other maintainers handle it without that person's involvement.

## Consequences

Maintainers may, depending on severity and pattern:

- Issue a private warning.
- Require a public correction.
- Remove offending content from project spaces.
- Temporarily or permanently ban a contributor from project spaces.
